# v1.9 - 2026-04-25

## 新增

- `stockpile` 数据库版本管理：新增 `schema_meta` / `schema_version`，并支持 `is_active` 字段迁移。
- `stockpile` 管理接口：新增 `/stockpile/inactive`、`/stockpile/changes`、`/stockpile/schema`。

## 修复

- 库存同步改为“当前系统导出镜像”语义，导出缺失条码会失效化，不再无限累积历史条码。
- 已失效条码重新导入或手工修正时会自动恢复为活跃状态，并记录 `reactivate` 审计日志。
- 月度汇总清理窗口改为“当前月在内最近 6 个自然月”，修复跨年场景 off-by-one。
- `pytest -q` 与 `python -m pytest -q` 入口统一可用，补齐 pytest 配置。
- JSON 文件读写改为短重试 + 原子替换，降低 Windows 文件锁导致的偶发失败。
- 测试按用例隔离目录，补齐 `stockpile` 失效记录、schema 迁移、月度保留窗口等回归测试。

- `pytest -q` -> `170 passed`
