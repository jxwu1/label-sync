# label-sync ERP 前端设计文档

## 产品概述

基于 Flask 的 ERP 库存标签处理系统，部署在公司内网。用户是仓库管理员和运营人员（2-5 人），日常通过 Chrome 桌面端访问。

**技术约束**：Flask + Jinja2 + Alpine.js + 原生 CSS，无构建步骤，无 npm/React/Vite。设计产出需要能直接翻译成 CSS class + HTML。

**设计方向**：Apple Design Language — 暗色优先，克制，圆角，弹簧动画，单色强调（蓝 #007AFF），需要同时出 Dark 和 Light 两套。

### 侧栏导航

- 10 个模块，每个有图标 + 中文标签 + 键盘快捷键
- 当前项高亮（accent 底色 + 白字）
- 折叠态只显示图标，hover 显示 tooltip
- 底部有折叠/展开按钮

### 顶栏

- 左：面包屑（系统名 / 当前模块）
- 中：搜索框（⌘K）
- 右：状态指示（IDLE/RUNNING/DONE）、实时时钟、主题切换按钮、语言标识

### 子信息条

- 单行，显示 SESSION ID、OPERATOR、STOCKPILE 记录数、系统在线时长

## 各模块详细功能

### 模块 01：标签处理（主页）

**用途**：上传扫描文件 → 三阶段自动处理 → 下载结果

**布局**：两栏 grid（左主区 + 右侧栏 340px）

**左栏面板：**

1. **文件投递** — 拖拽上传区（接受 .xlsx / .csv）+ 「开始处理」主按钮 + 「配置规则」「重放上次」次按钮
2. **处理管线** — 5 阶段进度条（PARSE → NORMALIZE → MATCH → AUDIT → COMMIT），每阶段有编号 + 标签 + 描述，当前阶段高亮+底部动画条，已完成阶段显示 ✓
3. **异常处理** — 处理后产生的异常卡片列表，每张卡左侧有严重度色条（HIGH 红 / MED 橙 / LOW 蓝），头部有严重度 pill 汇总

**右栏面板（纵向堆叠）：**

4. **文件队列** — 已选文件列表（文件类型图标 + 文件名 + 大小 + 状态 pill QUEUE/RUN/DONE/FAIL + 删除按钮），底部动作栏（上传/继续处理/下载结果/复制型号/清空）
5. **STOCKPILE 数据仓库** — 4 个 tab（状态/初始化/月度比对/搜索），状态 tab 有 2×2 stat 方格（总条数/上次 import/活跃/失效），带 ONLINE 绿色脉冲指示
6. **活动日志** — 实时日志流（mono 字体），LIVE 脉冲指示，日志按类型着色（ok 绿 / warn 橙 / err 红 / dc 蓝）

---

### 模块 02：标签查重

**用途**：检测扫描数据中的异常（空白/非法前缀/重复/空库位/负库存）

**布局**：

1. **5 列 stat 卡片** — 每类异常的计数（带颜色标识 error/warn）
2. **工具条** — 查询标签 + 扫描计数 + 异常计数 + 刷新/全部复制 CTA
3. **5 个折叠 panel** — 每类异常一个（whitespace / prefix / duplicate / empty location / negative stock），内含 mono 表格 + 复制按钮

---

### 模块 03：采购导入

**用途**：上传供应商 Excel → 解析 → 匹配数据库 → 管理新品条码

**布局**：

1. **上传区** — 拖拽 drop zone
2. **结果表格** — 条码/品名/供应商/价格列，状态 pill（match/new/check/off）
3. **新品管理** — 新条码列表，可编辑供应商/价格字段，保存/删除操作
4. **导入历史** — 底部时间线条，显示历史导入记录

---

### 模块 04：考勤台账

**用途**：员工考勤管理，日历视图 + 统计

**布局**：

1. **顶部工具栏** — 员工选择器 + 日期输入 + 操作按钮
2. **5 列 stat 卡片** — 总天数/出勤/缺勤/请假/特殊
3. **日历视图** — 7 列周网格，每个日期格有左色条（缺勤红/请假橙/特殊蓝），可点击打开日编辑 popover
4. **右侧员工列表** — 滚动 rail，每个员工卡有 2×2 mini stat

---

### 模块 05：货号历史

**用途**：按条码/货号查询完整生命周期数据

**布局**：3 个顶级 tab

**Tab 1 - 搜索：**
- 搜索输入 + 最近搜索 chip
- 查询结果展开为多个 panel：
  - CUR（当前状态）：库存/价格/供应商/来源
  - SLA（销售分析）：周销/月销/季销/趋势
  - PUR（采购记录）：历史进货表
  - EXT（扩展信息）：分类/标签/生命周期
  - RST（补货建议）：预测数据
  - TML（时间线图表）：52 周 SVG 折线图（双 Y 轴）
  - HIS（变更历史）：事件时间线卡片

**Tab 2 - 最近改动：**
- 批次选择器 + 5 个 stat 汇总 + 筛选 chip + 变更表格

**Tab 3 - 扫描记录：**
- 员工选择器 + 扫描记录列表

---

### 模块 06：数据质量

**用途**：监控数据异常（多库位/频繁翻转）

**布局**：

1. **信息 banner** — 只读提示 + 刷新按钮
2. **2 列 stat 卡片** — 多库位数 / 高频翻转数（带左色条）
3. **DQ-01 panel** — 多库位数据表
4. **DQ-02 panel** — 高频翻转数据表

---

### 模块 07：数据健康（进销存导入）

**用途**：导入采购/销售数据到系统 + 导入产品主数据

**布局**：

1. **健康概览** — 系统状态信息
2. **导入区** — radio 选择导入类型（采购/销售）+ 文件选择 + 预览/导入按钮
3. **产品 CSV 导入区** — 单独的产品主数据导入
4. **列映射面板** — 预览后显示，表格形式选择列对应关系
5. **导入结果** — 导入后显示汇总卡
6. **数据库状态** — stat 方格显示当前记录数
7. **历史记录** — 最近导入列表

---

### 模块 08：老外客人

**用途**：管理外国客户的月度应收账款

**布局**：

1. **控制条** — 月份选择器 + 刷新 + 下载 PDF + 添加记录按钮
2. **5 列 stat 卡片** — 总额/已收/未收/逾期/免单（带颜色 tone）
3. **记录表格** — 8 列（客户/欠款/税号/付款日/运费/状态/备注/操作），状态 pill（paid/unpaid/overdue/partial/free）
4. **新增/编辑 modal** — 表单：客户选择、月份、金额、税号、日期、备注

---

### 模块 11：补货决策

**用途**：基于销售预测的智能补货建议

**布局**：

1. **头部操作栏** — 导出/标记已订/跳过/捆绑按钮 + 已订筛选 chip + 刷新
2. **KPI 行** — 4 个指标按钮（总数/紧急/过量/无利润），点击可筛选
3. **供应商概览** — 按供应商分组的补货汇总
4. **筛选条** — 来源（FOREIGN/CN/未知）+ 视图（活跃/新品/全部）+ 覆盖周数滑块 + 供应商筛选 + 搜索
5. **数据表格** — 13 列可排序表格（复选/条码/品名/来源/供应商/在库/周销/覆盖周/建议量/紧迫分/利润/操作），支持多选批量操作

---

### 模块 10：互传

**用途**：A 端和 B 端之间文件传输 + 文字消息

**布局**：两栏 grid

1. **连接状态条** — 图标 + 标题 + ONLINE 脉冲指示
2. **左栏 - 文件传输** — 拖拽 drop zone + 文件列表（文件名/大小/类型标签/下载按钮）
3. **右栏 - 消息** — 消息气泡列表（分 A/B 端，带时间戳 + 删除按钮）+ 底部输入框（textarea + 复制/发送按钮）

## 通用 UI 组件

| 组件 | 说明 |
|------|------|
| Panel | 卡片容器：头部（编号 + 标题 + 副标题 + 右侧操作）+ 内容区 |
| Stat Card | 数字统计：大号 mono 数字 + 小号标签 + 可选颜色 tone |
| Table | 数据表：sticky 表头（大写/小号）+ 斑马纹 + hover 高亮 + mono 数字列 |
| Pill / Badge | 标签：圆角药丸形，支持 accent/warn/error/info/ghost 色调 |
| Button | 按钮：primary（accent 底 + 白字）/ ghost（透明 + 边框）/ danger |
| Drop Zone | 上传区：虚线边框 + 拖入高亮 + 图标 + 文字提示 |
| Tabs | 分段控制器：底部线条指示当前 tab |
| Modal | 弹窗：遮罩 + 居中卡片 + 头部/内容/底部按钮 |
| Empty State | 空态：居中文字提示 |
| Filter Bar | 筛选条：药丸形 chip + 搜索框 |

## 设计要求

1. **出图范围**：全部 10 个模块 + 全局框架（侧栏/顶栏），Dark 和 Light 各一套
2. **设计语言**：Apple HIG 风格 — 纯黑/纯白底，单色强调蓝 #007AFF，圆角 8-16px，多层低透明度阴影，系统字体栈
3. **信息密度**：这是内部 ERP 工具，表格密度优先于留白美感，但排版要干净
4. **中文排版**：所有 UI 文案都是中文，注意中文字体渲染（PingFang SC / Microsoft YaHei fallback）
5. **交互状态**：每个组件需要 default / hover / active / disabled / focus 状态
6. **响应式**：主要用于 1920×1080 桌面端，1100px 以下侧栏收起、表格横向滚动
7. **现有主题保留**：系统还有一套 "Terminal" 暗色主题（终端绿 #00ff95 + JetBrains Mono），需要能通过管理面板切换，但本次设计只做 Apple 风格
