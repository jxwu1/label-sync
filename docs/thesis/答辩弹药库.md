## A 组：方法选择（必问）

### Q1. 为什么不用 sklearn / statsmodels / 深度学习？是不会还是不需要？

【要点】间歇需求 + 短历史下，方法复杂度的边际收益为负，这是文献结论不是偷懒：
M5 竞赛中简单组合方法压制大量复杂模型；Croston 类方法 50 年没被实质超越。
本系统中位 SKU 每 3.6 周才卖一次（ADR-0002 F1），多数序列有效样本 < 50 个点，
深度模型没有施展空间。工程上：numpy 实现每个模型 ~30 行、可单测、可审计，
sklearn 引入的是依赖面和不可解释性，不是准确度。
【证据】`app/services/backtest.py` 全部模型合计 ~180 行；回测证明"最复杂的
HoltWinters"没有跑赢经验分位数（forecast eval 看板 models 对比）；CLAUDE.md
技术栈声明"numpy（预测算法 + 回测，不引 sklearn / statsmodels）"是设计决策。

### Q2. MAPE 0.814 这么差，这套预测还有什么用？

【要点】先纠正指标语义：间歇需求的点预测 MAPE 高是结构性的（多数周真值为 0
或 1-2 件，相对误差天然爆炸），文献早有共识。**本系统的设计立场是：补货消费
的不是点预测，是分位数**。分布校准（coverage）才是硬指标，点准度只是参考。
回测显示分布校准良好，这正是"点不准但分布可靠"的典型间歇需求画像。
【证据】ADR-0001 回测事实节："点预测仅略优于 naive；分布校准是这套系统真正
可靠的部分 → 策略设计应消费分位数而非点估计"。补货公式 S = Q_α(D_H)
（ADR-0001 D2）完全不消费点预测。

### Q3. 凭什么说 EmpiricalQuantile 这种"没有模型的模型"是合理选择？

【要点】三层：①统计上，经验分位数是分布的非参估计，在"分布形状未知、样本
有限、尾部重要"场景是最稳健选择；②实证上，4 baseline + 2 主力的 walk-forward
横评里它的 Cov@98 全面压制正态近似（0.972 vs 0.886-0.926）；③哲学上，
"复杂模型赢点准度 4% 但输覆盖 8 个百分点"时，选覆盖——因为补货消费尾部。
【证据】ADR-0002 D2 记录了 NaiveMean4W 点准 4% 但 Cov 差 8-9pp 的取舍；
ADR-0002 F2"象限内模型排序跨象限一致——赢家不随象限变"。

### Q4. Croston / SBA / TSB 各自的适用边界？为什么不用 TSB？

【要点】教科书答案：ADI > 1.32、CV² < 0.49 → Croston 区；SBA 加 (1−α/2)
修正 Croston 的正偏；TSB 的增量价值 = 每期更新需求概率，处理**需求消亡**。
本系统答案：①实证标定发现 99% SKU 落间歇侧，文献象限在这个组合上不分流
（ADR-0002 F1），所以路由按 sku_type 而非象限；②TSB 的适用场景（需求消亡）
被 dying 闸用更彻底的方式覆盖——直接不预测，而回测证明对 dying 强出预测
引入 +0.89 bias。TSB 是"渐退"，我们选"急停"，并记录了未来若做清仓残值
预测时 TSB 是首选。
【证据】ADR-0002 D4 全文；`tools/calibrate_model_routing.py` 可复现象限分布；
categorizer.py:56-62 dying 阈值（13/26 周类型感知）。

### Q5. 你的 SKU 分类（ADI×CV² / sku_type）阈值怎么定的？拍的吗？

【要点】两套分类分清楚：**销售形态**（retail/mixed/wholesale/dying）做路由，
阈值经 spike 脚本在退场标准 barcode 上验证；**生命周期**（new/seasonal/…）
不参与路由，且已确诊退化（97% unclassified）——主动交代这个，说明检测规则
已自动化（RL-11 巡检），重标定是已知 backlog。把"已知病灶 + 已建监控"讲成
工程成熟度的证据，而不是被动等问。
【证据】categorizer.py:50-54 阈值常量 + 注释（spike 验证）；ADR-0002 F4
（97.0% unclassified、seasonal=0 确诊）+ D5（不挂告警的理由：挂了就是永远红
的噪声）；`alerts._forecast_routing_degraded`（RL-11 已接 cron）。

## B 组：回测方法论（评委里有统计背景必问）

### Q6. 周分位数 × 8 为什么是错的？修正后推荐量为什么普遍下降？

【要点】分位数不可线性相加：i.i.d. 下 Q98(ΣD) ≈ 13μ + 2.05σ√13 ≪ 13×Q98(D)，
周间波动互相抵消（√N 律）。间歇序列最严重——周 p98 被单笔大单顶高，×8 后
高估 2-3 倍。修正 = bootstrap：有放回抽 H 个周值求和 ×2000 次取和分位数。
真实数据验证：新口径中位数是旧口径的 29.7%，即修正了 3.4 倍系统性高估。
【证据】ADR-0001 D5 + 实施状态注记（29.7%、1,011 个 SKU 缺货周剔除、在途
64k 件入 IP）；`forecast.horizon_quantile`（forecast.py:30，固定 seed=42）。

### Q7. bootstrap 假设周间独立，需求有自相关怎么办？

【要点】三段式：①承认——正自相关（连续旺季）会让和的真实尾部比 bootstrap 厚；
②对齐——生产模型 EmpiricalQuantile 本身就是 i.i.d. 假设，bootstrap 与模型
假设一致，不额外引入误差；③验证路径——walk-forward 框架可直接评估 H 周和
的经验覆盖率（coverage_h4），这是该修正的量化验收。若已实施（审计修复 #4），
直接报数字；未实施，承认是进行中的验收项。
【证据】ADR-0001 D5"记录的已知限制"段落原文；审计 MT-3。

### Q8. 缺货周怎么处理？为什么剔除而不是插值/修正？

【要点】缺货周的 0 是删失观测不是真实需求，入训练会形成死亡螺旋：预测被拉低
→ 补货更少 → 更缺货——专门惩罚畅销品。选**剔除**因为：对 i.i.d. 模型，剔除
等价于"对有货周的需求分布取样"，统计上干净；插值引入人造值污染分位数；
基于仿真的需求重建是正确的升级路径但复杂度高一个量级（论文二档路线图）。
诚实声明覆盖范围：库存快照 2026-05-20 起，修正只能向前生效，随数据积累自动增强。
【证据】ADR-0001 D7 全文；`stockout.exclude_stockout_weeks`（"当缺失，不填 0
不插值"）；`forecast_output.stockout_weeks_excluded` 字段供置信分层消费；
RL-3 测试 `test_rl3_exclusion_raises_forecast`（剔除后分位数只升不降）。

### Q9. 你的回测有没有 look-ahead 泄漏？

【要点】**主动交代，变守为攻**：我们对框架做过系统性泄漏审计，发现分类与
清洗层的时间无界查询（分类 as_of 与 IQR 阈值读到 end_date 之后的事件）。
影响评估：已发表 run 的 end_date≈执行日，未来数据基本不存在，引用数字不受
实质影响；但框架做历史时点回测时泄漏成立，修复方案（fetch 加 as_of 上界）
已排期/已完成。能当场画出泄漏传播路径（分类→准入→清洗阈值→训练序列）本身
就是加分项。
【证据】审计 LK-1/LK-2 全文（categorizer.py:326/290、forecast_data.py:184）；
修复 PR（答辩前查状态）。

### Q10. 测试窗重叠，你的样本数虚高，差异显著吗？

【要点】承认设计：步进 1 周 + 测试窗 4 周 → 每个真实周最多进 4 条记录，记录
间相关。因此①论文不做基于记录数的朴素显著性检验；②模型比较以 SKU 级聚合
（每 SKU 一个 MASE）为单位，SKU 间近独立，配对比较可用 Wilcoxon signed-rank；
③h=1 子集（无重叠）做过敏感性对照（若审计修复 #9 已做，报结论一致性）。
【证据】审计 MT-2；`walk_forward_backtest` records 结构（backtest.py:199-216）。

### Q11. coverage 0.972 里有多少是零需求周白送的？

【要点】这是 coverage 指标在间歇需求上的已知特性：p98 ≥ 0 恒覆盖 actual=0。
回答分两层：①模型横评不受影响（所有模型同享零周覆盖，差异来自非零周）；
②绝对值解读已补条件覆盖率 coverage_p98_nonzero（仅 actual>0 周），论文双列
并示——非零覆盖必然更低，但那才是"来真需求时分位数够不够用"的指标。
（若修复 #3 未完成：承认"在做"，给出预期方向。）
【证据】审计 MT-5；backtest.py:248-252。

### Q12. 模型是在同一份数据上选的又在同一份数据上报分数，过拟合了吧？

【要点】承认 selection-conditioned 偏差存在，但量化它的空间：候选只有 6 个、
模型间结构差异大（不是在调同族超参）、walk-forward 多窗平均本身抑制
snooping。补救：时间分窗 confirmation——用旧窗口复现选择、用 2026-03 后的
滚动新数据只跑已选模型报确认分数；快照数据每周积累，这是持续验证机制而非
一次性补丁。
【证据】审计 SN-1；BASELINES 共 6 项（backtest.py:271-278）。

### Q13. MASE 是标准定义吗？

【要点】（修复前）诚实承认：分母用了测试记录序列的 lag-1 差分而非训练集
in-sample naive MAE，绝对值不可与文献对标，但分母与模型无关，6 模型相对排序
有效；已按标准定义修正并重跑（修复 #2），论文引用修正后数字。**绝对不要
试图蒙混"是标准 MASE"**——这是一行代码能验的。
【证据】审计 MT-1；backtest.py:236-245。

## C 组：补货策略（业务向评委必问）

### Q14. 为什么选 (R,S) 周期盘点而不是教科书更常见的 (s,Q)/(s,S)？

【要点】策略参数化必须匹配**库存可见性的物理事实**：scraper 每周一抓快照，
库存位置只有周粒度。连续盘点制需要实时库存位置，在这套系统里是虚构——选
(s,Q) 等于给不存在的数据建模。R=1 周不是设计选择，是数据节奏的事实，写死
并文档化。反震荡触发阈值（缺口 ≥ max(1 中包, 0.25S) 或断货）承担了教科书
s 的工程职能，同时解决间歇 SKU 上 s/S 退化重合的边界。
【证据】ADR-0001 D1 + D6；`restock_calc._churn_gate`（restock_calc.py:268）。

### Q15. 安全库存公式呢？z·σ·√L 去哪了？

【要点】被实证淘汰：教科书公式假设需求正态，本组合 99% 间歇，回测里正态
近似的 Cov@98 只有 0.886-0.926，被经验分位数 0.972 全面压制。S = Q_α(D_H)
取经验分位数后，Q_α(D_H) − μ·H 就是隐式安全库存——不是没有安全库存，
是不走参数化公式。这是"用回测证据替代教科书假设"的直接案例。
【证据】ADR-0001 D2 + 回测事实节 + 备选方案表"正态近似安全库存公式：否决"。

### Q16. lead time 只有 1 个到货样本，4 周是怎么敢定的？

【要点】三段：①现状诚实——purchase_orders 2026-05-27 才上线，历史 ERP 只有
到货日没有下单日，lead time 分布当前不可得；②先验有依据——4 周是用户按
实际供应链（中国采购→希腊海运）校准的经验值，进配置不硬编码；③自我修正
机制——arrival_date 样本 ≥20 单自动切经验分布取 p90，且解释为什么 p90 不是
均值：L 进保护期，偏短的代价（缺货）远大于偏长的代价（多备 1-2 周），右偏
分布下均值低估风险。设计的可贵处在于"知道自己不知道 + 数据到位自动升级"。
【证据】ADR-0001 D4 + 审计事实 1（11 单、1 个 arrival 样本）；lead time
先验配置 + 经验 p90 自动切换实现（#41 实施 D4）。

### Q17. 推荐量上线后普遍下降，用户凭什么信系统没坏？

【要点】下降是**回到统计正确水位**，不是回归：三项修正方向一致——①在途
64,165 件首次进入库存位置（不再重复下单）；②bootstrap 替代 ×8 线性放大
（修正 3.4 倍高估）；③缺货周剔除（畅销缺货 SKU 预测回升，这项是涨的）。
配套的信任机制：sanity 闸只标记不截断（异常量醒目展示而非静默吞掉）、
p50/p98 双档保留人工判断空间、置信分层诚实标注 missing_backtest。
【证据】ADR-0001 后果节 + 实施状态注记（29.7%、459 SKU 在途、1,011 SKU
缺货剔除）；RL-6"不自动截断（截断掩盖上游 bug）"。

### Q18. 单笔大单怎么处理？剔除大单不会把批发需求剔没了吗？

【要点】按 SKU 形态分流，不一刀切：retail_dominant 用 doc 级 IQR 剔除
（median + 3·IQR，阈值用 median/IQR 而非均值——防 wholesale 大单污染阈值
本身）；mixed 加客户类型过滤；**wholesale_only 不剔大单**——大单就是它们的
需求本体，剔了等于把信号当噪声。这是"清洗策略必须依赖需求形态"的核心论点。
注意术语：实际实现是 doc 级 IQR 剔除，不是周级 winsorization（plan 早期
提过 winsorize，后被 IQR 方案取代——答辩别说做了 winsorization）。
【证据】forecast_data.base_demand_view 分流规则（forecast_data.py:165-171）；
`is_bulk_order`（median+k·IQR，forecast_data.py:135）；ADR-0002 D1
"wholesale 不剔大单：大单就是它们的需求本体"；审计 ST-2（winsorize 死代码）。

### Q20. 这套系统的预测准确度提升，最终换来了什么业务价值？

【要点】用三个落地数字 + 一个机制回答，不谈抽象准确度：①在途扣减消除了

【证据】ADR-0001 实施状态注记；restock skip 抑制 feature（1379200）；

## 通用答辩姿态

1. **每个设计决策都讲成"备选方案 + 否决理由"**——ADR 已经是这个结构，直接搬。
2. **已知缺陷主动交代 + 给出检测/修复机制**，比被问出来强一个数量级；
   本系统的卖点之一就是"退化有自动告警"（RL-9/10/11）。
3. **被问到没准备的数字**：给方向和机制，不现编数字——"具体值需要查 run
   元数据，但方向是 X，因为机制是 Y"。
4. **区分"统计正确"与"业务正确"**：很多取舍（p90 lead time、p98 服务水平、
   只标记不截断）的依据是代价不对称，不是统计最优。这是论文的成熟度信号。
