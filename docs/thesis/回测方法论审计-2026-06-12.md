# 回测方法论统计审计（对抗性）

> **Date**: 2026-06-12
> **作者**: Fable 5（判断层产出；修复实施由后续 plan 执行）
> **审计对象**: `app/services/backtest.py`、`app/utils/forecast_data.py`、
> `app/utils/categorizer.py`、`app/services/forecast_eval.py`、`app/services/stockout.py`
> **立场**: 站在答辩评委/审稿人角度，专找 look-ahead 泄漏、data snooping、
> 幸存者偏差、指标定义缺陷。每条 finding 给代码证据、对已发表结论的影响评估、修复建议。
> **配套**: `docs/thesis/答辩弹药库.md`（把本审计的"承认 + 缓解"转成答辩话术）

## 总评

框架的**骨架是对的**：expanding-window walk-forward、训练/测试时间切分、
多模型同序列横评、view 变体记录进 run 元数据 —— 这些比多数课程项目强。
真正的问题集中在三处：**①清洗与分类层的时间无界查询**（回测时构成 look-ahead）、
**②MASE 分母偏离标准定义**（绝对值不可与文献对标）、**③coverage@p98 在间歇
序列上被零周机械抬高**（0.972 的含金量需要拆解）。三处都可修，且修复不动框架。

对已引用结论（run 44：MAPE 0.814 / MASE 0.978 / Cov@98 0.972）的影响分级：
- **泄漏类（LK-\*）影响小**：run 44 的 `end_date` ≈ 执行日，"未来"基本不存在，
  无界查询读不到多少未来数据。**但框架做历史时点回测（固定过去的 end_date）时泄漏成立**。
- **指标类（MT-\*）影响实质**：MASE 绝对值、coverage 绝对值都不能按教科书语义解读。
  模型间**相对排序**（同一指标同一数据横评）仍然成立 —— ADR-0002 的路由决策不受动摇。

## 一、Look-ahead 泄漏

### LK-1【高】分类与清洗的时间无界查询

- **位置**:
  - `categorizer._fetch_sku_doc_net_qty`（categorizer.py:326）—— 无 `event_at <= as_of` 过滤
  - `categorizer._fetch_last_sale_at`（categorizer.py:290）—— 同上
  - `forecast_data.base_demand_view:184` —— IQR 大单阈值用上面的无界查询，
    docstring 自陈"窗口外也算, 阈值更稳"（生产视角合理，回测视角 = 泄漏）
- **机理**: 回测调 `classify_sku_type(as_of=end_date)`，dying 判定用 as_of，
  但 doc-net 形态分类（retail/wholesale/mixed）和 IQR 阈值消费**全历史含
  end_date 之后**的事件。三个传染路径：
  1. SKU 形态分类被未来销售改写 → 路由/准入（`series=None` 与否）由未来决定；
  2. 大单剔除阈值 `median + 3·IQR` 含未来 doc → 每个训练截点的"清洗后序列"
     都掺了未来分布信息；
  3. `as_of` 在过去时，之后复活的 SKU 不会被判 dying（被未来销售"救活"）。
- **对已发表结论的影响**: run 44 `end_date`≈执行日 → 泄漏面≈0，**已引用数字不必撤回**。
  但 ADR-0002 的标定（wholesale 子集对比）若复跑历史时点，结论需复核。
- **修复**（给 Opus，半天）: 两个 fetch 函数加 `as_of` 参数（`event_at < as_of+1d`），
  生产调用传 today（行为不变），回测/标定调用传 end_date。加一条回归测试：
  构造"as_of 之后才有大单"的 SKU，断言分类与阈值不受其影响。

### LK-2【中】序列清洗在 walk-forward 循环外做一次

- **位置**: `backtest._build_series`（backtest.py:281）→ `walk_forward_backtest`
- **机理**: 清洗（大单剔除、缺货剔除、分类）对整条序列做一次，然后各训练截点
  切片。即使 LK-1 修了（阈值只看 ≤end_date），对 start 较早的训练窗，阈值仍然
  用了"该截点之后、end_date 之前"的数据。严格做法是每个截点用 `series[:cut]`
  重算阈值。
- **影响评估**: 二阶效应。IQR 阈值随样本量收敛较快，全期阈值 vs 截点阈值的差
  主要影响序列前段（样本少时）。**承认 + 声明**即可，不值得为它把回测复杂度
  乘 O(n)：论文里写"清洗阈值按回测期末估计（quasi-ex-ante），逐截点重估的
  消融留作 future work"。
- **修复**: 文档声明（论文 + plan），不改代码。

## 二、指标定义

### MT-1【高】MASE 分母不是标准定义

- **位置**: `backtest.mase`（backtest.py:236-245）+ `run_backtest_for_sku` 的
  records 展平（backtest.py:346-348）
- **机理**: 标准 MASE（Hyndman & Koehler 2006）分母 = **训练集内** one-step
  naive 预测的 MAE。这里分母 = 把 (start, h) 双重循环展平的 actual 序列做
  lag-1 差分。该序列里同一真实周重复最多 4 次，且相邻元素存在 h=4→h=1 的
  跨窗跳变（如 …w16, w17, **w15**, w16…）。分母既不是训练集 naive MAE，
  也不是测试集 naive MAE，是无明确统计语义的混合量。
- **影响评估**: "MASE 0.978 < 1 = 跑赢 naive"这句话**不能对文献口径下断言**。
  同一分母下 6 个模型的相对排序不受影响（分母与模型无关），所以模型选择结论
  （EmpiricalQuantile / 路由表）站得住。`beats_naive_pct`（forecast_eval.py:194）
  同病。
- **修复**（给 Opus，1 天）: walk_forward 每窗用 `train` 的 in-sample lag-1
  MAE 做该窗 records 的分母，按窗聚合后再平均；或全局用整条序列 in-sample
  naive MAE。修后重跑 6 模型 base_demand run，论文引用新数字并标注定义。
  旧数字若要保留，改名 rMAE 并写明定义。

### MT-2【中】重叠测试窗 → 样本重复加权、n 虚高

- **位置**: `walk_forward_backtest`（backtest.py:199-216）：start 步进 1、
  window_test=4 → 每个真实周最多进 4 条 record。
- **机理**: 中段周在 metrics 里权重是端点周的 4 倍；records 高度相关 →
  以 record 数当 n 做任何显著性检验都会假阳性；`mean_actual` 等聚合也被
  中段加权。
- **影响评估**: 对点估计（中位 MASE 等）影响温和（加权≈平滑），对"差异是否
  显著"类问题致命 —— 目前论文没做显著性检验，所以是**潜在雷**而非现伤。
- **修复**: 答辩口径：声明 records 相关性、不做朴素 t 检验。若要显著性：
  按 SKU 聚合后对模型差做 Wilcoxon signed-rank（SKU 间近独立），或 h=1 子集
  （无重叠）单独算一套指标做敏感性对照。后者半天可加。

### MT-3【中】回测验证的是周边际分位，生产消费的是 H 周和分位

- **位置**: `coverage_p98` 评估 `predict()` 的周 p98（backtest.py:248-252）；
  生产补货消费 `horizon_quantile` 的 H 周**和**分位（restock_calc / forecast.py:30）
- **机理**: ADR-0001 D5 把"周分位 ×N"修成 bootstrap 和分位，并自己写明
  "留给回测验证：walk-forward 框架可直接评估 H 周和的覆盖率" —— **这个验收
  还没做**。当前 Cov@98=0.972 证明的是周粒度校准，不能直接背书 8 周和的校准
  （i.i.d. 成立时二者一致，但 i.i.d. 正是待验假设：正自相关会让和的真实尾部
  比 bootstrap 厚）。
- **修复**（给 Opus，1 天，**论文二档价值**）: walk_forward 每窗加一条
  horizon record：`sum(test[0:4])` vs `horizon_quantile(train, 4, 0.98)`，
  聚合成 coverage_h4。这一项做完，ADR-0001 D5 的链路就闭环了，论文多一张
  "周覆盖 vs 4 周和覆盖"对比表。

### MT-4【低】MAPE 剔零周 —— 间歇序列上系统性偏样

- **位置**: `backtest.mape`（backtest.py:220-226），docstring 已自陈。
- **机理**: 间歇序列大半周为 0，剔除后 MAPE 只在非零周上算 → 样本偏向
  "有需求的周"，且对偏高预测惩罚不对称。行业已知缺陷，这正是引入 MASE 的原因。
- **修复**: 不修代码。论文写明 MAPE 仅作直觉参考、决策依据是
  MASE（修正版）+ coverage。

### MT-5【低→论文层面中】coverage@p98 被零周机械抬高

- **位置**: `coverage_p98`（backtest.py:248-252）：`actual <= p98` 计入覆盖，
  p98 ≥ 0 恒成立 → **每个 actual=0 的周都是免费覆盖**。
- **机理**: 间歇序列零周占比高，0.972 里有相当一部分来自 0 ≤ p98 的恒真。
  评委一句"你的覆盖率有多少来自零需求周？"就能把 0.972 问出水分。
- **影响评估**: 不影响模型横评（同病同医），影响绝对值的解读与论文表述。
- **修复**（给 Opus，半天）: 加 `coverage_p98_nonzero`（仅 actual>0 的周），
  dashboard 和论文双列并示。预期非零覆盖会明显低于 0.972 —— 这不丢人，
  提前算出来写进论文比被当场问出来强一个数量级。

### MT-6【低】代码卫生三件

- `bias` 空输入返回 0.0 而非 None（backtest.py:229-233）—— 0 是合法偏差值，
  语义混淆;
- `zip(..., strict=False)` 在 mape/bias/mase/coverage —— 长度错配静默截断，
  应 strict=True 让 bug 炸出来;
- `compare_run_pair` 的中位数自实现（backtest.py:537-541）与 numpy 重复。
- **修复**: 顺手修，不单开 PR。

## 三、选择偏差 / data snooping

### SN-1【中】模型选择与报告在同一数据集上

- **机理**: 6 个模型跑同一份历史 → 选赢家 EmpiricalQuantile → 论文引用赢家
  在**同一份数据**上的分数。经典 selection-conditioned 乐观偏差。同理：RL-4
  收缩参数 (30 周, 1.5×)、CrostonSBA α=0.1、路由准入"非零周 ≥5"都没有独立
  验证窗。
- **影响评估**: 候选只有 6 个、彼此差异大、walk-forward 本身多窗平均 →
  snooping 空间有限，结论方向不会翻。但论文必须声明，且有便宜的补救。
- **修复**（给 Opus，半天）: 时间分窗 confirmation run —— 用截至 2026-03 的数据
  做 selection（复现路由决策），用 2026-03→06 做 confirmation（只跑已选模型，
  报告这一段的分数）。新快照数据每周还在积累，这个设计还能持续滚动。

### SN-2【中】入选过滤用全期信息 → 幸存者偏差

- **位置**: `run_backtest_for_sku`：`nonzero < min_weeks(20)` 在整条序列上数
  （backtest.py:338-339）。
- **机理**: SKU 是否入选回测由**含测试期**的非零周数决定 —— "后来卖起来了"
  的 SKU 被纳入，测试期死掉的被排除。结论覆盖面被静默缩小到"全期活跃" SKU，
  而补货风险最大的稀疏/濒死 SKU 恰好不在覆盖内。
- **修复**: 过滤条件改为只数**训练可用段**（前 `n - window_test` 周）的非零周；
  论文口径写明"结论适用于训练期非零 ≥20 周的 SKU（占比 X%），稀疏 SKU 的
  预测置信由分层机制（missing_backtest → low）兜底"。

### SN-3【低】view="all" 的 dying/unclassified 甩出依赖 LK-1 的无界分类

- **位置**: backtest.py:305-307。
- **机理**: "强出预测 dying 引入 +0.89 bias"的结论（ADR-0002 D4 引用）是在
  无界分类下测的；LK-1 修复后应复跑确认数字仍成立（方向几乎不可能翻，
  但引用值会变）。
- **修复**: 跟 LK-1 修复同一 PR 复跑，更新 ADR 引用值。

## 六、审计声明（论文可直接引用的限制段落素材）

1. 回测为 expanding-window walk-forward，步进 1 周、测试窗 4 周；测试窗重叠，
   记录间存在相关性，故不做基于记录数的朴素显著性检验，模型比较以 SKU 级
   聚合统计为准。
2. 需求清洗（异常大单剔除）阈值按回测期末样本估计（quasi-ex-ante）；
   时间严格的逐截点重估留作 future work，预期影响集中在序列前段。
3. MAPE 在间歇需求上仅作参考；模型选择依据为 MASE（修正定义）与
   coverage@p98（含条件覆盖率）。
4. 模型族候选共 6 个，选择与报告基于同一历史区间；时间外推确认窗
   （2026-03 后滚动数据）作为持续验证机制。
