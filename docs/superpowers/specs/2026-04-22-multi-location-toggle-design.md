# 多库位 Toggle 多选设计

**Date:** 2026-04-22
**Status:** Approved
**Scope:** `static/js/index-dup.js` (单文件改动)

## 背景

当前多库位冲突 UI 中，店面和仓库各只有两种选项：选一个 或 保留全部。
用户需要任意组合选择（如三选二），现有单选模式无法满足。

## 设计

### 交互规则

- 每个库位按钮改为 toggle（点击切换选中/未选中）
- 所有库位数量场景统一使用 toggle 模式（包括 2 个库位）
- 「保留全部」按钮保留，点击后全选
- 至少选中 1 个库位才能确认

### 视觉状态

- 选中：橙色高亮（`dup-active`，与现有一致）
- 未选中：默认灰色
- 保留全部选中时：绿色（`dup-keep-all`，与现有一致）

### 数据结构变更

- `p2DupSel[barcode][type]` 从 `string | "__all__"` 改为 `Set<string>`
- 保留全部 = Set 包含全部 unique 值
- `composeDupParts` 从 Set 取值拼接，用 `/` 连接

### 确认按钮启用条件

- 店面和仓库各自：有库位选项时至少选中 1 个
- 无库位选项的一侧自动跳过

### 后端

无需改动。最终提交的 resolution 仍为 `/` 拼接的字符串。

## 影响范围

| 文件 | 改动 |
|------|------|
| `static/js/index-dup.js` | 选择状态改 Set、toggle 逻辑、compose 逻辑、高亮逻辑 |
