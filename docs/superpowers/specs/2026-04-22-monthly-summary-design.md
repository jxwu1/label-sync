# 采购订单月度财务总结设计

**Date:** 2026-04-22
**Status:** Approved
**Scope:** 新功能

## 背景

采购订单目前是无状态处理（上传→处理→导出）。用户需要按月汇总每笔采购的财务数据，生成 PDF 报表。

## 数据采集

在采购订单导出 ZIP **之前**，弹出表单收集：

| 字段 | 来源 | 类型 |
|------|------|------|
| 供应商名称 | 用户输入 | 文本 |
| 总价 | 系统计算（价格×数量求和），用户确认 | 数字（€） |
| 税金 | 用户输入 | 数字（€） |
| 加税总价 | 系统计算（总价 + 税金） | 数字（€） |
| 开票日期 | 用户输入 | 日期 |

## 数据持久化

- 存储方式：JSON 文件，按自然月命名（如 `monthly_summary/2026-04.json`）
- 每条记录结构：

```json
{
  "supplier_name": "ABC贸易",
  "total_price": 12000.00,
  "tax": 1560.00,
  "total_with_tax": 13560.00,
  "invoice_date": "2026-04-15",
  "created_at": "2026-04-15T14:30:00"
}
```

- 自然月周期：UI 只展示当月数据，旧文件保留不删

## PDF 报表

- 每笔采购一个独立小表格
- 布局：

```
┌──────────────────────────────┐
│ 供应商：ABC贸易              │
│ 开票日期：2026-04-15         │
├──────────┬───────────────────┤
│ 总价     │ €12,000.00        │
│ 税金     │ €1,560.00         │
│ 加税总价 │ €13,560.00        │
└──────────┴───────────────────┘
```

- 货币单位：欧元（€）
- PDF 生成库：reportlab

## 前端交互

### 导出前表单

在现有导出流程中，点击导出后先弹出模态框：
- 供应商名称输入框
- 总价显示（自动计算，可修改）
- 税金输入框
- 加税总价显示（自动计算，不可修改）
- 开票日期选择器
- 「确认并导出」按钮：保存记录 + 触发原有 ZIP 导出
- 「跳过，直接导出」按钮：不保存记录，直接导出 ZIP

### 月度总结区域

采购订单页面底部：
- 显示当月已有记录条数
- 「下载月度总结 PDF」按钮
